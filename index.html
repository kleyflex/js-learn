Правила написания
чистого JavaScript кода

Основные правила

01

Используйте понятные имена переменных

Название переменной должно четко отражать, какие данные она хранит. Избегайте односимвольных имен (кроме счетчиков в циклах) и общих слов типа data, info, temp.

Плохо 
let a = 25;
let x = "Иван";
let data = [1, 2, 3];
Хорошо 
let userAge = 25;
let userName = "Иван";
let productPrices = [1, 2, 3];
Совет
Используйте camelCase для имен переменных: первое слово с маленькой буквы, остальные — с большой. Например: userAge, totalPrice, isActive.
02

Правильно выбирайте const и let

Используйте const для значений, которые не будут переназначаться, и let для изменяемых переменных. Забудьте про var — он устарел и создает проблемы.

Правильно 
const PI = 3.14159;
const userName = "Мария";

let counter = 0;
counter = counter + 1;
Правило
По умолчанию всегда используйте const. Если позже понадобится изменить значение, тогда замените на let.
03

Форматируйте код последовательно

Всегда используйте одинаковые отступы (2 или 4 пробела), добавляйте пробелы вокруг операторов и после запятых. Последовательность важнее личных предпочтений.

Плохо 
let sum=a+b;
let numbers=[1,2,3,4,5];
if(x>5){
console.log("больше 5");
}
Хорошо 
let sum = a + b;
let numbers = [1, 2, 3, 4, 5];

if (x > 5) {
  console.log("больше 5");
}
04

Пишите осмысленные комментарии

Комментарии должны объяснять почему вы делаете что-то, а не что делает код. Хороший код сам себя документирует через понятные имена.

Плохой комментарий 
// Увеличиваем i на 1
i = i + 1;
Хороший комментарий 
// Пропускаем заголовок таблицы при обработке данных
for (let i = 1; i < rows.length; i++) {
  processRow(rows[i]);
}
05

Используйте правильные типы данных

Выбирайте тип данных, соответствующий задаче. Не храните числа в строках, используйте булевы значения для флагов, выбирайте массивы для списков однородных данных.

Плохо 
let age = "25";           // число хранится как строка
let isActive = "true";    // булево значение как строка
let firstName = ["Иван"]; // одно значение в массиве
Хорошо 
let age = 25;
let isActive = true;
let firstName = "Иван";
let studentNames = ["Иван", "Мария", "Петр"];
06

Делайте циклы читаемыми

В циклах используйте осмысленные имена для переменных-счетчиков, когда это улучшает понимание. Добавляйте пустые строки для визуального разделения блоков кода.

Хорошо 
// Для простых счетчиков i, j, k — стандарт
for (let i = 0; i < students.length; i++) {
  console.log(students[i]);
}

// Для сложной логики — описательные имена
for (let studentIndex = 0; studentIndex < students.length; studentIndex++) {
  const currentStudent = students[studentIndex];
  const nextStudent = students[studentIndex + 1];
  
  compareStudents(currentStudent, nextStudent);
}
07

Один оператор — одна строка

Не размещайте несколько операторов на одной строке. Это ухудшает читаемость и усложняет отладку.

Плохо 
let x = 5; let y = 10; let z = x + y;
Хорошо 
let x = 5;
let y = 10;
let z = x + y;
08

Работайте с массивами корректно

Используйте length вместо магических чисел, проверяйте границы массива, используйте понятные имена для массивов (множественное число).

Плохо 
let arr = [1, 2, 3, 4, 5];

for (let i = 0; i < 5; i++) {  // магическое число
  console.log(arr[i]);
}
Хорошо 
let numbers = [1, 2, 3, 4, 5];

for (let i = 0; i < numbers.length; i++) {
  console.log(numbers[i]);
}

Общие принципы

DRY — Don't Repeat Yourself

Не повторяйте один и тот же код. Если видите дублирование — это сигнал, что нужна функция (мы скоро до них дойдем).

KISS — Keep It Simple, Stupid

Пишите простой и понятный код. Сложность должна появляться только там, где она действительно нужна для решения задачи.

Пишите код для людей

Помните: код читается гораздо чаще, чем пишется. Через месяц даже вы сами будете благодарны за понятный код, который написали сегодня.

Эти правила — основа профессиональной разработки.

Следуйте им с первого дня, и они станут вашей второй натурой.

Удачи в обучении!
